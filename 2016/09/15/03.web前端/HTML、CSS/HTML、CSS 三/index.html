内联块元素的特性，及不同类型元素之间的转换（display属性）
  • 内联块元素：
    • 也叫行内块元素，是新增的元素类型，现有元素没有归于此类别的，img和input元素的行为类似这种元素，但是也归类于内联元素，我们可以用display属性将块元素或者内联元素转化成这种元素。
  • 内联块元素的特性：
    • 支持全部样式（和内联元素唯一一点的区别）
    • 如果没有设置宽高，宽高由内容决定
    • 盒子并在一行
    • 代码换行，盒子会产生间距
    • 子元素是内联块元素，父元素可以用text-align属性设置子元素水平对齐方式。
  • display属性（基本只用于三种元素类型之间的转换）
    • 1、none 元素隐藏且不占位置（相当于将当前的标签元素取消）
    • 2、block 元素以块元素显示
    • 3、inline 元素以内联元素显示
    • 4、inline-block 元素以内联块元素显示
浮动属性
  • 浮动的作用：
    • 对元素进行浮动排列，让元素既有块元素、内联元素的特性，又可以进行无间隙的在同一行内进行排列
  • 浮动的特性：
    • 1、浮动元素有左浮动(float:left)和右浮动(float:right)两种
    • 2、浮动的元素会向左或向右浮动，碰到父元素边界、其他元素才停下来（并没有完全脱离文档流）
    • 3、相邻浮动的块元素可以并在一行，超出父级宽度就换行
    • 4、浮动让行内元素或块元素转化为有浮动特性的行内块元素(此时不会有行内块元素间隙问题)
    • 5、块元素和浮动元素的文档流之间是不存在布局联系的，块元素会按照没有浮动元素相同的位置布局，浮动元素会覆盖块元素（块元素的文字部分会绕过浮动的元素显示）。
    • 6、父元素如果没有设置尺寸(一般是高度不设置)，父元素内整体浮动的子元素无法撑开父元素，父元素需要清除浮动
    • 7、浮动的元素不能设置居中，只能够左浮动或者右浮动；
    • 8、当两个嵌套浮动元素的父元素相邻的时候，如果一个父元素中的浮动元素溢出，会影响另外一个父元素中的浮动元素的布局，即一个父元素中的浮动元素如果超出父元素边界到另外一个父元素中，那吗另外一个父元素中的元素可能会和溢出的浮动元素排列在同一行，向后排列， 这种现象可以设置父元素的overflow：hidden解决；
      • 注意：使用内联块元素和浮动的属性和使用内联元素时候，尽量让同一行的元素的属性相同，因为对其的基线的原因，可能会出现对齐方式不同的情况，导致元素在同一行不能对齐；
  • 清除浮动的方法：
    • 1、父级上增加属性overflow：hidden
    • 2、在最后一个子元素的后面加一个空的div，给它样式属性 clear:both（不推荐）
      • clear：  both/left/right   为定义两侧不相邻带有浮动元素的属性
    • 3、使用成熟的父类清浮动样式类，clearfix   （同时解决了margin-top塌陷）
      • .clearfix:after,.clearfix:before{ content: "";display: table;}       “清除浮动，解决margin塌陷”
      • .clearfix:after{ clear:both;}      “清除浮动”
      • .clearfix{zoom:1;}    “兼容ie浏览器”
元素的定位属性
  • 文档流 
    • 文档流，是指盒子按照html标签编写的顺序依次从上到下，从左到右排列，块元素占一行，行内元素在一行之内从左到右排列，先写的先排列，后写的排在后面，每个盒子都占据自己的位置。
  • 元素的定位：
    • 元素的定位属性
      • 使用元素的属性，将元素定位在相对父元素或者浏览器某个位置，我们可以用css的position属性来设置元素的定位类型，
    • position的设置项如下：
      • relative（相对定位，文档位置保留，元素类型不变） 生成相对定位元素，元素所占据的文档流的位置保留，元素本身相对自身原位置进行偏移。
      • absolute （绝对定位，文档位置不保留，元素已经变成了内联块元素）生成绝对定位元素，元素脱离文档流，不占据文档流的位置，可以理解为漂浮在文档流的上方，相对于上一个设置了相对定位的父级元素来进行定位，如果找不到，则相对于body元素进行定位，body如果也没有相对关系，那么会根据浏览器定位。
      • fixed （相对于浏览器定位，文档位置不保留，元素已经变成了内联块元素）生成固定定位元素，元素脱离文档流，不占据文档流的位置，可以理解为漂浮在文档流的上方，相对于浏览器窗口进行定位。
        • 当元素被设置为fix定位的时候，已经是基于浏览器来定位，可以根据浏览器的大小的百分比来设置元素的宽高（此时已经不是根据父元素来参考），设定的元素宽高为浏览器的百分比大小；
      • static 默认值，没有定位，元素出现在正常的文档流中，相当于取消定位属性或者不设置定位属性。
      • 备注：当对元素进行绝对定位时，如果不给元素top/left等属性，元素的位置默认为父元素的（0,0）位置；
    • 定位元素的偏移（设置元素基于选择标准的偏移）
      • 定位的元素还需要用left、right、top或者bottom来设置相对于参照元素的偏移量。
        • 备注：
          • 一般情况下，定位的基准的反方向为正取向；一般的基准选择为（top-left）（right-bottom）
          • 设置偏移可以用  数字%，来表示元素的偏移量，表示基于父元素的比例大小；
    • 定位元素层级 （用来控制定位元素显示的优先级，一般从1000开始设置）
      • 定位元素是浮动在正常的文档流之上的，可以用z-index属性来设置元素的层级
    • 备注（关于绝对定位和固定定位的元素）：
      • 绝对定位和固定定位的块元素和行内元素会自动转化为行内块元素

表单标签及相关属性和样式
  • html表单标签
    • 表单用于搜集不同类型的用户输入，表单由不同类型的标签组成
  • 表单的标签及其用法（在访问服务器的时候，url后面的参数就是通过表单input来进行提交的，局部刷新的时候是使用ajax进行提交的，提交数值的格式为键值对，键值对的形式为：'name', 'value'）：
    • <form>标签 定义整体的表单区域，（块元素，成对使用）
      • action属性：定义表单数据提交地址，一般为服务器地址；
      • method属性：定义表单提交的方式，一般有“get”方式和“post”方式，与请求方式对应；
        • get  内容拼接在域名后面； post  内容不会显示，会将信息包含在请求体中；
        • 提交的内容是以键值对的方式提交的，键是input的name属性，value是输入或者选择的内容；
    • <label>标签 为表单元素定义文字标注  （内联元素，成对使用，经常和文本输入合用）
      • for属性：可以关联input标签的id，这样点击lable标签的内容的时候，会选中input标签；
    • <input>标签 定义通用的表单元素(内联元素，单独使用)
      • type属性：定义元素类型
        • type="text" 定义单行文本输入框，经常搭配属性:
          • placeholder属性：当输入框中没有内容的时候会进行提示；
          • value属性：为输入框中输入的内容；
        • type="password" 定义密码输入框
          • value属性：为输入框中输入的内容；
        • type="radio" 定义单选框，经常搭配属性：
          • name属性：会让name相同的单选框具有互斥的效果；
          • value属性：选择对应单选框时提交的内容；
          • checked属性：当框被勾选时，有具有checked属性；
        • type="checkbox" 定义复选框，经常搭配的属性：
          • name属性：name相同的的元素会公用一个键进行提交；
          • value属性：当选择了对应的多选框的时候提交的内容；
          • checked属性：当框被勾选时，有具有checked属性；
        • type="file" 定义上传文件
        • type="submit" 定义提交按钮，经常搭配的属性：
          • value属性：属性值会直接显示在提交框内；
        • type="reset" 定义重置按钮，会充值所有标签的而输入，经常搭配的属性：
          • value属性：属性值会直接显示在重置框内；
        • type="button" 定义一个普通按钮
      • value属性： 定义表单元素的值
      • name属性： 定义表单元素的名称，此名称是提交数据时的键名
    • <textarea>标签 定义多行文本输入框 （内联元素，成对使用）；
      • name属性：定义提交数据时候的键；
      • cols属性：定义最少占多少列的宽度；
      • rows属性：定义最少占多少行的高度；
      • placeholder属性：当输入框中没有内容的时候会进行提示
    • <select>标签 定义下拉表单元素，需要配合option标签使用（内联元素，成对使用）
      • name属性：定义提交数据时候的键；
    • <option>标签 与<select>标签配合，定义下拉表单元素中的选项（内联元素，成对使用）；
      • value：定义选择对应的opt标签的时候提交的数据；
  • CSS表单的样式属性
    • outline 设置input框获得焦点时，是否显示凸显的框线，一般设置为没有,比如：
      • outline:none;
表格属性及其相关属性和样式
  • 表格标签：定义一个由一个表格标签，其中有多个行 、列标签组成；
  • 表格标签及其用法：
    • <table>标签 （定义一个table块元素）：声明一个表格
    • <tr>标签：定义表格中的一行
    • <td>和<th>标签：定义一行中的一个单元格，td代表普通单元格，th表示表头单元格（带有默认属性文字加粗，和文字居中等），它们的常用属性如下：
      • width  设置单元格的宽度，不设置会根据包含的内容来按照一定规则分配；
      • height  设置单元格的高度，不设置会根据包含的内容来按照一定规则分配；
      • colspan 设置单元格水平合并，设置值是数值
        • 例子：<th colspan="5" height="16%">基本情况</th>
      • rowspan 设置单元格垂直合并，设置值是数值
        • 例子：<td rowspan="5" width="15%">性 别</td>
    • 注意：
      • 在设置表格单元格的高度和宽度时候，不能通过样式来设置，需要通过td/th的属性width/height 来设置，在设置每列的宽度的时候，使用具有列的数量最全的行来设置，一般为内容th的第一行；
  • 表格标签的样式属性：
    • border-collapse 设置表格的边线合并，如：
      • border-collapse:collapse;
        • 默认属性值为：separate ；边线不会重合，具有一定的间距；
    • vertical-align  表格中的内容垂直对位方式，如：
      • vertical-align: middle；
        • 默认值为：top； 有top、middle、bottom 属性； 注意vertical 属性只能在表格中使用，不能应用在其他的元素样式中；

CSS权重
  • CSS权重的介绍：
    • CSS权重指的是样式的优先级，有两条或多条样式作用于一个元素，权重高的那条样式对元素起作用,权重相同的，后写的样式会覆盖前面写的样式。
  • 权重的等级：
    • 1、!important，加在样式属性值后 “；”前，权重值为 10000
    • 2、内联样式，如：style=””，权重值为1000
    • 3、ID选择器，如：#content，权重值为100
    • 4、类，伪类，如：.content、:hover 权重值为10
    • 5、标签选择器，如：div、p 权重值为1
  • 权重的计算方式
    • 一个元素的样式属性的权重是对样式权重值进行加和，元素会使用对应权重总和最大的样式属性；
